The Failure Mode Nobody Catches

Automated underwriting AI fails most dangerously not when it is obviously wrong — but when it is confidently wrong on low-quality data. A model that calls a patient low-risk with 95% confidence on 40% missing data is running on priors, not evidence. Most systems have no mechanism to detect this. MedRisk is built around one.

Layer 1

Data Quality Scoring

Every patient record is scored for completeness and reliability before inference, so the decision pipeline knows how much weight the input can actually bear.

Layer 2

Plausible-But-Wrong Detection

Cases where model confidence exceeds what the input data can support are flagged and escalated — automating the cases that are safe, surfacing the ones that need a human.

Layer 3

Regulated by Design

Conformal validation, SHAP explanations, and EU AI Act–aligned governance and audit trails are built in, not bolted on after the fact.

What It Changes

The point is not higher model accuracy. It is knowing which predictions to trust — which is what lets you safely automate the routine majority and reserve human underwriters for the cases that genuinely need them.

MedRisk is a production platform built as an independent project. The platform code, model internals, validation methodology, and data adapters are private. This page is an overview; the implementation is available under a pilot, licence, or consulting engagement — not as open source.
