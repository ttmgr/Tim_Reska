What it is

Signal, not a feed

The coding-agent ecosystem moves faster than any changelog can be read by hand: models, CLIs, IDE plugins, the Model Context Protocol, and a steady stream of arXiv papers all ship weekly. The radar collapses that into a short, curated digest aimed at one question — what changed that should change how I work? Items lead with an action title and the practical implication, and every claim carries a link to its primary source (a release page, a spec commit, or the paper itself).

How an issue is built

Parallel scan, deduplicated, curated

  1. 1
    Fan out.

    Parallel research agents sweep the six clusters at once, each scoped to its own set of primary sources.

  2. 2
    Deduplicate against the ledger.

    An append-only "seen" ledger is read first, so each issue reports deltas rather than re-surfacing last week's items.

  3. 3
    Curate to a working stack.

    What survives is filtered to what actually affects day-to-day agent-assisted engineering — not every release, only the ones that change a decision.

  4. 4
    Write action titles with sources.

    Every item leads with the takeaway and the practical implication, and links to its primary source for verification.

Research — two papers dismantle two common safety habits

  • SWE-Mutation: LLMs cannot reliably generate test suites — a mutation-based benchmark where even strong models reach only ~36% detection, and realistic mutations drop detection from 71% to 40%. The strongest evidence yet that "an agent wrote a passing test ⇒ the behavior is correct" is unsafe; pair generated tests with a mutation step or an independent oracle. arXiv:2605.22175 · 2026-05-21
  • Articulate but Wrong: self-review fails on code modernization — across 1,980 Python 2→3 migrations and 11 production models, 39.7% silently change semantics and 31.7% of those failures pass the model's own self-review. It does not improve with capability tier, which kills "ask the model to review its own diff" as a safety net for refactors. arXiv:2605.21537 · 2026-05-20
  • Co-ReAct: per-step rubrics beat free-form ReAct — a rubric generator injects per-step rubrics into a ReAct loop, with consistent gains on research and coding benchmarks across model sizes. A practical pattern for tightening subagents. arXiv:2605.23590 · 2026-05-25
  • The Impact of AI Coding Assistants — a 6-month longitudinal study — 82% of developers spend less time writing code as work shifts to verification and "supervisory engineering"; perceived productivity stays stable (84% report gains) while developer-experience scores degrade (27% poor vs 14% baseline) — the experience tax to budget for. arXiv:2605.23135 · 2026-05-22
