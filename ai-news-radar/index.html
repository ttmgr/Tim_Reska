What it is

Signal, not a feed

The coding-agent ecosystem moves faster than any changelog can be read by hand: models, CLIs, IDE plugins, the Model Context Protocol, and a steady stream of arXiv papers all ship weekly. The radar collapses that into a short, curated digest aimed at one question — what changed that should change how I work? Items lead with an action title and the practical implication, and every claim carries a link to its primary source (a release page, a spec commit, or the paper itself).

How an issue is built

Parallel scan, deduplicated, curated

  1. 1
    Fan out.

    Parallel research agents sweep the six clusters at once, each scoped to its own set of primary sources.

  2. 2
    Deduplicate against the ledger.

    An append-only "seen" ledger is read first, so each issue reports deltas rather than re-surfacing last week's items.

  3. 3
    Curate to a working stack.

    What survives is filtered to what actually affects day-to-day agent-assisted engineering — not every release, only the ones that change a decision.

  4. 4
    Write action titles with sources.

    Every item leads with the takeaway and the practical implication, and links to its primary source for verification.

Research — agents still misjudge when a task is "done"

  • DeployBench: agents deploy real artifacts at only 7.8–51% — 51 research-artifact deployment tasks spanning multi-language toolchains, system deps beyond containers (GPU/CUDA, kernel config) and legacy compatibility, each verified by a hidden pipeline that runs the paper's experiment. Four SOTA models with OpenHands pass 7.8–51.0%; most failures come from wrongly judging the task complete. arXiv:2606.05238 · 2026-06-03
  • Search-time contamination inflates deep-research scores — letting an agent search the web during evaluation leaks benchmark answers and inflates scores by ~4%. Eval-hygiene takeaway: sandbox the network when scoring web-enabled agents, and discount leaderboard numbers that don't. arXiv:2606.05241 · 2026-06-03
  • Agent Planning Benchmark (APB) — multimodal agents show systematic weakness in extended planning, tool robustness, and calibrated task refusal; a diagnostic for where multi-step agents break before you wire them into anything load-bearing. arXiv:2606.04874 · 2026-06-03
  • ADK Arena: framework choice is a real cost/quality lever — across 51 agent frameworks, no single kit dominates and generation cost spans a 5.6× range for comparable work. Treat framework selection as an explicit decision, not a wash. arXiv:2606.05548 · 2026-06-04
