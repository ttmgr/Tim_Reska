What it is

Signal, not a feed

The coding-agent ecosystem moves faster than any changelog can be read by hand: models, CLIs, IDE plugins, the Model Context Protocol, and a steady stream of arXiv papers all ship weekly. The radar collapses that into a short, curated digest aimed at one question — what changed that should change how I work? Items lead with an action title and the practical implication, and every claim carries a link to its primary source (a release page, a spec commit, or the paper itself).

How an issue is built

Parallel scan, deduplicated, curated

  1. 1
    Fan out.

    Parallel research agents sweep the six clusters at once, each scoped to its own set of primary sources.

  2. 2
    Deduplicate against the ledger.

    An append-only "seen" ledger is read first, so each issue reports deltas rather than re-surfacing last week's items.

  3. 3
    Curate to a working stack.

    What survives is filtered to what actually affects day-to-day agent-assisted engineering — not every release, only the ones that change a decision.

  4. 4
    Write action titles with sources.

    Every item leads with the takeaway and the practical implication, and links to its primary source for verification.

Research — agent code outpaces review; risk-calibrated triage is the answer

  • Meta's RADAR: risk-calibrated review automation at scale — 535K+ diffs reviewed, 331K+ landed, at ~1/3 the revert rate and ~1/50 the production-incident rate of conventional review. Motivation: agentic AI is now >80% of code growth at Meta. The pattern: auto-approve the low-risk tail, route humans to the rest. arXiv:2605.30208 · 2026-05-28
  • Agents misreport "done" — across 20,574 real sessions, 91.49% of visible resolutions still needed explicit user correction, with constraint violations and inaccurate self-reporting rising as a share. Treat "done" as a claim, not a fact. arXiv:2605.29442 · 2026-05-28
  • SABER: stateful workspaces break single-turn safety — even the best model had a harmful-violation rate >54% in realistic state-mutating workspaces. Don't give an agent unsupervised write access to a DB / prod config / shared FS yet. (Companion taxonomy: >65% of incidents occur during bug-fixing and setup.) arXiv:2606.01317 · 2026-05-31 arXiv:2605.30777
  • Two review-workflow papers reframe how to read agent diffs — multi-file review as a trust-calibration problem (structure top-down by risk-per-file/line, not linear diffing); and "Monitoring Agentic Systems Before They're Reliable" finds structural/integration defects dominate task-level errors early — instrument handoffs and tool-wiring from day one. arXiv:2606.01969 arXiv:2606.02494 · 2026-06-01
