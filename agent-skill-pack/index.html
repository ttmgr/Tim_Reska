Agent Skill Pack

A coding-agent skill pack for One Health genomics

A machine-readable, inspectable layer over the GenomicsForOneHealth pipelines. It lets LLM-assisted coding agents (Codex, Claude Code, Cursor, Continue) route a sequencing project to the right documented workflow, validate inputs, build commands only from declared templates, parse outputs, run sanity checks, and write audit logs — without inventing commands, parameters, tools, or databases.

Python standard library (+ PyYAML); no LLM required

What it is

A workflow-orchestration layer, not a chatbot demo

The pack is designed for inspectability and reproducibility, and it is useful even with no LLM runtime: the hooks and the eval runner are plain Python you can call directly. The local repository is the source of truth — current public Oxford Nanopore and EPI2ME references are used only for comparison and never override the local workflows. Every curated command must cite the repository file it was extracted from.

The hook pipeline

Five hooks around every skill

Standard library only, except command_builder.load_skill_yaml, which uses PyYAML.

preflight.py

Input checks

Verifies inputs, files, directories, databases, and command availability before anything is built.

command_builder.py

Build, never execute

Loads a skill, validates parameters, builds command strings. Refuses undeclared parameters, reports missing required ones, and shell-quotes values to prevent injection.

parsers.py

Structured outputs

Parses NanoStat, Kraken2 reports, AMRFinderPlus tables, VCF, and generic TSV into dicts — never raising on bad input.

validation.py

Sanity flags

Flags low yield, low N50, low classification rate, contamination, missing AMR hits, empty output, and missing databases or commands.

audit.py

Traceable log

Writes a JSON audit log with timestamp, inputs, parameters, commands, results, flags, source files, and external references.

Coverage

Each skill declares its domain, supported inputs, version-pinned tools, required databases, parameters, ordered command templates (extracted verbatim from the repository), hook references, outputs, caveats, failure modes, and the source files it derives from.

Quick start

Runs offline, no LLM and no bioinformatics tools needed

# inspect a skill
python -c "from agent_skills.hooks import command_builder as cb; \
import json; print(json.dumps(cb.load_skill_yaml('agent_skills/skills/cre_plasmid_clustering.yaml')['skill']['display_name']))"

# run the offline benchmark suite
python agent_skills/evals/run_benchmarks.py

Caveats

Suggestions, not conclusions

All outputs from this pack are suggestions, not biological, clinical, regulatory, or diagnostic conclusions. Commands are built, not executed; the human remains responsible for running them and interpreting results.
