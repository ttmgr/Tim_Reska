Prompting Patterns

Reusable structures for clearer instructions, better context, and easier review.

The Core Template

  • Goal: what outcome do I need?
  • Audience: who will read or use it?
  • Inputs: what material should the model rely on?
  • Constraints: length, tone, exclusions, risk boundaries.
  • Output: exact structure to return.

Ask for Useful Friction

  • List assumptions before answering.
  • Ask clarifying questions only if needed.
  • Flag uncertainty and missing data.
  • Separate facts from interpretation.
  • Give a verification checklist with the answer.

Context Pack

  • Task background: one paragraph.
  • Source material: paste only what is allowed and relevant.
  • Definitions: terms, abbreviations, audience knowledge.
  • Examples: one good example beats five vague instructions.
  • Rubric: what a strong answer must include.

When Output Is Weak

  • Do not just say 'make it better'. Name what is wrong.
  • Ask for three alternatives when you are exploring.
  • Ask for a revision against your rubric.
  • Use short iterations. Fix one problem at a time.

Four Provider-Neutral Prompt Patterns

Pattern Use When Starter
Transform You have material and need a new form. Rewrite the text below for [audience] as [format]. Keep [constraints].
Compare You need differences, risks, or options. Compare A and B on [criteria]. Return a table and a recommendation.
Tutor You want to learn without outsourcing thinking. Teach me [topic]. Ask me 3 questions, then explain gaps.
Reviewer You have a draft and need critique. Review this against [rubric]. Flag unclear, unsupported, risky parts.
